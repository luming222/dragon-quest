# 草系统重做方案

## 当前问题判断

当前草的问题，不只是在“贴图本身”，而是这几件事叠在一起：

1. 草叶平面偏宽偏高，贴图容易显得被拉长。
2. 单丛交叉面偏少，远看像纸片。
3. 土国草量偏密，容易把地表压满，看不出“干、硬、风化”的感觉。
4. 着色只有基础染色，缺少根部偏暗、顶端偏干的层次。

## 这次已做的第一轮修正

1. 调整草的几何默认值
   - 更窄：减少贴图横向假感
   - 更矮：减少过度拉伸
   - 更多交叉面：提升体积感

2. 调整草的 shader
   - 改成 `depth_prepass_alpha`
   - 增加根部/顶部颜色层次
   - 风摆只主要作用于草尖
   - 让草底更压暗，减少纸片感

3. 调整土国区域默认参数
   - 降低草密度
   - 草色改成更干、更偏土黄的调子

## 第二轮建议

如果第一轮后仍觉得“贴图味”重，下一步建议按这个顺序继续：

### A. 先换草贴图，不先换系统

优先比较：

- `res://assets/textures/grass_alpha.png`
- `res://assets/textures/grass_candidate.png`

目标不是“更清晰”，而是：

- 轮廓更碎
- 根部更集中
- 顶端更轻
- alpha 边缘更干净

### B. 做土国专用草参数

土国草不应和木国草共用同一感觉。

建议后续加两组 profile：

- `earth_dry_grass`
- `wood_lush_grass`

土国草特征：

- 更稀
- 更短
- 更黄
- 摆动更小
- 更靠近石块/坡脚聚集

### C. 草和地表做联动

现在草是独立散布，下一步可以加规则：

- 石道中央少草
- 残碑周围少草
- 坡脚和边缘多草
- 土坡顶部少、背风面多

这样会比单纯换贴图更有效。

## 结论

这套草系统不用推翻。

正确方向是：

- 保留程序草 + MultiMesh
- 先修几何比例和 shader
- 再做土国专用草 profile
- 最后才决定是否替换草贴图
