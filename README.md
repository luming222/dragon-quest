# Dragon Quest

一个基于 Godot 开发的单人动作 RPG 原型项目，当前聚焦于五行主题战斗、关卡竖切验证，以及 PC / Mobile 共用一套内容与交互框架。

项目目标不是做一个单纯的数值砍杀原型，而是验证一条更明确的战斗体验路线：

- 以拼刀 / Clash 窗口为核心节奏
- 通过技能释放与时机命中触发共鸣
- 用五行国家建立战斗身份、关卡气质与叙事差异

## 项目概述

- 类型：单人动作 RPG
- 视角：2.5D 斜视角 / Quarter View
- 引擎：Godot 4.6
- 当前状态：玩法原型与竖切验证阶段

项目核心方向包括：

- 以拼刀 / Clash 时机为中心的近战战斗
- 技能与 Clash 窗口结合触发共鸣
- 五行国家与元素身份差异化设计
- 同时兼顾 PC 与移动端操作适配

## 当前内容

仓库目前包含以下原型模块：

- 玩家、敌人、伙伴等基础角色逻辑
- 战斗管理与技能表现
- HUD 与触屏操作 UI
- 关卡、地形与运行管理脚本
- 五行相关设计文档与竖切路线稿

## 当前原型重点

当前版本主要在验证以下问题：

- Clash 战斗是否足够形成独特记忆点
- 土国竖切能否在 10 分钟内建立世界观、伙伴关系与 Boss 命题
- PC 与移动端是否能共享同一套核心玩法框架
- 风格化东方气质是否能与五行玩法方向统一起来

## 运行方式

1. 使用 Godot 4.6 打开项目目录。
2. 导入并打开 `project.godot`。
3. 运行主场景 `scenes/Main.tscn`，或直接从编辑器运行项目。

## PC 默认键位

- 移动：W / A / S / D
- 轻攻击：鼠标左键
- 重攻击：鼠标右键
- 闪避：Space
- 招架：F
- 技能 1 / 2 / 3：1 / 2 / 3

## 目录结构

- `scenes/`：主要场景资源
- `scripts/`：角色与主流程脚本
- `combat/`：战斗管理逻辑
- `skills/`：技能行为脚本
- `ui/`：HUD 与触控 UI
- `world/`：关卡、地形、运行管理
- `data/`：元素、词缀、装备等数据配置
- `docs/`：设计文档、视觉方向、竖切方案

## 核心文档

- `docs/design.md`：整体玩法与系统设计草案
- `docs/earth_nation_intro.md`：土国开场内容稿
- `docs/earth_nation_vertical_slice.md`：土国 10 分钟竖切路线
- `docs/visual_direction.md`：整体视觉风格方向
- `docs/grass_rework_plan.md`：草地与场景表现调整计划

## 设计方向

当前设计强调：

- 先承受，再理解，再反击
- 五行不是单纯属性换皮，而是节奏、反馈与战斗身份的差异
- 世界、伙伴关系与 Boss 设计共同服务整体旅途气质

后续将继续补全：

- 国家主题竖切内容
- 角色与场景视觉统一
- 技能演化与数据驱动配置
- 更完整的战斗与成长循环

## 开发路线

接下来的开发重点会集中在：

- 打磨主战斗循环与敌人节奏差异
- 补强土国竖切中的场景表达与伙伴互动
- 统一角色、环境与特效的视觉语言
- 逐步把原型脚本和配置迁移为更稳定的数据驱动结构
