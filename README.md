# Dragon Quest

一个基于 Godot 开发的单人动作 RPG 原型项目，当前聚焦于五行主题战斗、关卡竖切验证，以及 PC / Mobile 共用一套内容与交互框架。

## 项目概述

- 类型：单人动作 RPG
- 视角：2.5D 斜视角 / Quarter View
- 引擎：Godot 4.6
- 当前状态：玩法原型与竖切验证阶段

项目核心方向包括：

- 以拼刀 / Clash 时机为中心的近战战斗
- 技能与 Clash 窗口结合触发共鸣
- 五行国家与元素身份差异化设计
- 同时兼顾 PC 与移动端操作适配

## 当前内容

仓库目前包含以下原型模块：

- 玩家、敌人、伙伴等基础角色逻辑
- 战斗管理与技能表现
- HUD 与触屏操作 UI
- 关卡、地形与运行管理脚本
- 五行相关设计文档与竖切路线稿

## 运行方式

1. 使用 Godot 4.6 打开项目目录。
2. 导入并打开 `project.godot`。
3. 运行主场景 `scenes/Main.tscn`，或直接从编辑器运行项目。

## PC 默认键位

- 移动：W / A / S / D
- 轻攻击：鼠标左键
- 重攻击：鼠标右键
- 闪避：Space
- 招架：F
- 技能 1 / 2 / 3：1 / 2 / 3

## 目录结构

- `scenes/`：主要场景资源
- `scripts/`：角色与主流程脚本
- `combat/`：战斗管理逻辑
- `skills/`：技能行为脚本
- `ui/`：HUD 与触控 UI
- `world/`：关卡、地形、运行管理
- `data/`：元素、词缀、装备等数据配置
- `docs/`：设计文档、视觉方向、竖切方案

## 设计方向

当前设计强调：

- 先承受，再理解，再反击
- 五行不是单纯属性换皮，而是节奏、反馈与战斗身份的差异
- 世界、伙伴关系与 Boss 设计共同服务整体旅途气质

后续将继续补全：

- 国家主题竖切内容
- 角色与场景视觉统一
- 技能演化与数据驱动配置
- 更完整的战斗与成长循环
